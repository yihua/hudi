hudi-agent-gateway {{ .Chart.Version }} deployed.

  provider : {{ .Values.gateway.provider }} ({{ .Values.gateway.model }})
  trino    : {{ .Values.trino.host }}:{{ .Values.trino.port }} (catalog {{ .Values.trino.catalog }})
  mcp      : {{ if .Values.mcpEnabled }}enabled at /mcp{{ else }}disabled{{ end }}

Try it:
  kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "hudi-agent-gateway.fullname" . }} 8000:{{ .Values.service.port }}

  # chat UI
  open http://localhost:8000/ui/

  # agent chat API
  curl -X POST localhost:8000/v1/chat -H 'Content-Type: application/json' \
    -d '{"message": "How many trips per city?", "session_id": "s1"}'

  # connect an MCP client (e.g. Claude Code)
  claude mcp add --transport http hudi-lakehouse http://localhost:8000/mcp/

{{- if and (eq .Values.gateway.provider "ollama") (contains "host.minikube.internal" .Values.gateway.ollama.baseUrl) }}

NOTE: the gateway expects Ollama on the HOST machine. If it only binds
loopback, restart it reachable from the cluster:
  OLLAMA_HOST=0.0.0.0 ollama serve
{{- end }}

Readiness (kubectl -n {{ .Release.Namespace }} get pods) stays NotReady until
both Trino and the LLM endpoint are reachable -- `curl localhost:8000/ready`
shows per-dependency status.
